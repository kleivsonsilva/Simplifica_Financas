Autenticação

ℹ️ Todas as rotas marcadas com 🔒 exigem o header Authorization: Bearer <token>. O token JWT é obtido via POST /api/auth/login e tem validade de 24 horas. Tokens revogados por logout são bloqueados via blacklist no Redis. 
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

O payload do token contém: sub (user_id UUID), email, nome, iat e exp.

⏱️

Rate Limits

Rate limiting é aplicado por IP no API Gateway usando Redis. Respostas 429 Too Many Requests são retornadas ao exceder o limite.

RotaLimiteJanela
POST /api/auth/registro10 req60 s
POST /api/auth/login20 req60 s
GET/POST /api/transacoes120 req60 s
Demais rotas autenticadas60 req60 s

Request Body (JSON)

CampoTipoReq.Descrição
nomestring✱Nome completo

  "modo_interface": "simples"

Respostas

201 Created
{
  "message": "Usuário criado com sucesso",
  "token": "eyJ...",
  "usuario": {
    "id": "uuid",
    "nome": "João Silva",
    "email": "joao@email.com",
    "modo_interface": "simples",
    "data_criacao": "2026-06-10T..."
  }
}
409 Conflict
{ "error": "Email já cadastrado" }
POST /api/auth/login Autentica credenciais e retorna token JWT ▼

Request Body (JSON)

{
  "email": "joao@email.com",
  "senha": "senha123"
}

Respostas

200 OK
{
  "token": "eyJ...",
  "usuario": {
    "id": "uuid",
    "nome": "João Silva",
    "email": "joao@email.com",
    "modo_interface": "simples"
  }
}
401 Unauthorized
{ "error": "Credenciais inválidas" }
POST /api/auth/logout Invalida o token atual (blacklist) 🔒 ▼

Notas

Não requer body. O token do header é adicionado à blacklist no Redis por 24h. Sessão do usuário é removida.

Respostas

200 OK
{ "message": "Logout realizado com sucesso" }

POST /api/transacoes Cria uma nova transação (receita ou despesa) 🔒 ▼

Request Body (JSON)

CampoTipoReq.Descrição
tipostring✱receita ou despesa
valorfloat✱Valor positivo em R$
descricaostring✱Descrição breve
categoriastring–Padrão: Outros
datastring–YYYY-MM-DD. Padrão: hoje

Categorias Despesa

Alimentação · Moradia · Transporte · Saúde · Educação · Lazer · Vestuário · Serviços · Estoque · Outros

Categorias Receita

Salário · Vendas · Freelance · Investimentos · Aposentadoria · Aluguel · Outros

Resposta 201

{
  "message": "Transação criada com sucesso",
  "id": "mongo_id"
}
⚡ Ao criar uma transação, um evento é publicado na fila transacao_criada do RabbitMQ. O Goal Service e o Notification Service reagem assincronamente.

Metas

Microsserviço goal-service · MongoDB · RabbitMQ (consumer). Metas são atualizadas automaticamente quando uma receita é registrada — a meta com menor percentual de conclusão recebe o valor.

GET /api/metas Lista metas do usuário 🔒 ▼

Query Parameters

ParamPadrãoValores
statusativaativa · concluida · cancelada · todas

Resposta 200

[{
  "id": "mongo_id",
  "titulo": "Viagem",
  "valor_alvo": 3000.00,
  "valor_atual": 1200.00,
  "percentual": 40.0,
  "status": "ativa",
  "cor": "#6366F1"
}]
POST /api/metas Cria uma nova meta financeira 🔒 ▼

Request Body (JSON)

CampoTipoReq.Descrição
titulostring✱Nome da meta
valor_alvofloat✱Valor objetivo em R$
descricaostring–Detalhes opcionais
categoriastring–Padrão: Outros
data_iniciostring–YYYY-MM-DD. Padrão: hoje
data_limitestring–YYYY-MM-DD
corstring–Hex color. Padrão: #6366F1

Resposta 201

{
  "message": "Meta criada com sucesso",
  "id": "mongo_id"
}
PUT /api/metas/{meta_id} Edita campos de uma meta existente 🔒 ▼

Campos editáveis (todos opcionais)

titulo · descricao · valor_alvo · data_limite · categoria · cor

POST /api/metas/{meta_id}/deposito Adiciona valor manualmente a uma meta ativa 🔒 ▼

Request Body (JSON)

{ "valor": 200.00 }

Resposta 200

{
  "message": "Depósito realizado",
  "novo_valor": 1400.00
}

Se o novo valor ≥ valor_alvo, o status é atualizado automaticamente para concluida.

POST /api/metas/{meta_id}/cancelar Cancela uma meta (status → cancelada) 🔒 ▼
200 OK
{ "message": "Meta cancelada" }
DELETE /api/metas/{meta_id} Remove permanentemente uma meta 🔒 ▼
200 OK
{ "message": "Meta excluída com sucesso" }

Notificações

Microsserviço notification-service · Redis (lista LIFO, até 50 itens por usuário) · RabbitMQ (consumer). Notificações são geradas automaticamente a cada transação criada ou excluída. Despesas acima de R$ 1.000 geram alerta especial.

GET /api/notificacoes Lista as últimas 20 notificações do usuário 🔒 ▼
200 OK
{
  "notificacoes": [
    {
      "mensagem": "⚠️ Despesa alta registrada: R$ 1500.00 em Moradia",
      "tipo": "alerta",
      "data": "2026-06-10T10:23:00",
      "lida": false
    }
  ],
  "total": 1
}
// tipos: "info" | "sucesso" | "alerta"
DELETE /api/notificacoes/limpar Remove todas as notificações do usuário 🔒 ▼
200 OK
{ "message": "Notificações limpas" }

O endpoint de health do Gateway agrega o status de todos os microsserviços.

GET /health Status de todos os serviços (sem autenticação) ▼
200 OK
{
  "gateway": "ok",
  "status": "ok",
  "services": {
    "auth":         "ok",
    "transaction":  "ok",
    "goal":         "ok",
    "report":       "ok",
    "notification": "ok"
  }
}
// status pode ser: "ok" | "degraded" | "down"
